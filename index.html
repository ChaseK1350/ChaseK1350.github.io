Enhancement One: Software Design and Engineering

This enhancement improves the structure, usability, and maintainability of the To-Do List application. Updates include refined sorting logic, improved user interface elements, and better code organization.

Download Enhancement One ZIP

View Enhancement One Narrative

Artifact Description

The artifact I selected for this enhancement is the Android-based To-Do List application that I originally developed during CS-360: Mobile Architecture and Programming. The application allows users to create, update, and manage tasks through a mobile interface. It was initially created as a foundational project to demonstrate basic Android development concepts such as activities, layouts, and user interaction. For this enhancement, I revisited and expanded the application to improve both its functionality and overall design.

Reason for Selection

I chose this artifact for my ePortfolio because it clearly demonstrates my growth in software development, particularly in mobile application design and architecture. The original version of the app was functional but fairly simple. Through this enhancement, I was able to build on that foundation by adding more advanced features such as task categories using a Spinner, persistent storage through a database, improved user interface elements, and input validation. These additions showcase my ability to refine an existing codebase, improve usability, and implement more structured and maintainable solutions. The inclusion of database integration and better UI design especially highlights my ability to think beyond basic functionality and focus on real-world application needs.

Course Outcomes Alignment

This enhancement aligns well with the course outcomes I originally selected, specifically outcomes 3 and 4. I was able to design and evaluate a more complete computing solution by adding features that improve how users interact with the app and how data is managed. At the same time, I applied practical tools and techniques commonly used in mobile development, such as event handling, data persistence, and interface design. At this point, I do not feel the need to adjust my outcome-coverage plans, as the work competed clearly demonstrates these objectives.

Learning Experience

Working on this enhancement was a very valuable and fun learning experience. One of the biggest takeaways for me was understanding how to extend an existing application rather than starting from scratch. This required me to think more carefully about code organization, compatibility with existing features, and how changes in one part of the app could impact others. I also gained more experience working with user input validation and ensuring that the app handles errors gracefully, which is something that is easy to overlook in smaller projects.

Challenges and Problem Solving

There were definitely some challenges along the way. One issue I ran into was getting the Spinner for task categories to function correctly, as it initially did not display any options. Troubleshooting that problem helped me better understand how adapters and UI components work together in Android. Additionally, integrating the database required careful attention to how data is stored and retrieved so that the app remained stable and responsive.

Conclusion

Overall, I think this enhancement helped me move beyond basic app development and focus more on creating a polished, user-friendly, and maintainable application. It reinforced the importance of planning, testing often, and iterating on existing code, all of which are essential skills in real-world software development.

Enhancement Two: Algorithms and Data Structures

This enhancement focuses on optimizing task management through improved algorithms and efficient data handling techniques, resulting in better organization and performance.

Download Enhancement Two ZIP

View Enhancement Two Narrative

Artifact Description

The artifact I selected for this enhancement is the same mobile To-Do List application originally developed in CS-360. This application was initially created as a basic task manager that allowed users to add, edit, and delete tasks in a simple, unstructured list without advanced organization or retrieval features. For this enhancement, the application was significantly improved to incorporate more advanced algorithms and data structures, transforming it into a more efficient and user-friendly task management system.

Reason for Selection

I chose this artifact for my ePortfolio because it provided a strong foundation to demonstrate growth in algorithm design and data structure implementation. The original project was relatively simple, which made it an ideal candidate for meaningful enhancement. The updated version showcases several important skills like sorting algorithms, searching techniques, and data modeling. More specifically, the implementation of multi-level sorting using custom comparators highlights my ability to design and apply comparison-based algorithms. Also, the integration of a real-time linear search demonstrates an understanding of search algorithms and their practical application in user interfaces. Expanding the data structure by adding a priority field and updating the database schema further demonstrates my ability to adapt and optimize data models for new functionality. These improvements collectively show a deeper understanding of how algorithms and data structures directly impact performance and usability.

Enhancements Implemented

The artifact was significantly improved from the original form. Tasks are now dynamically sorted based on priority and name, ensuring that the most important items are always visible first. A more advanced three-level sorting system was introduced in the main list and the “Show All Tasks” view, organizing tasks by category, priority, and name. In addition, a real-time search feature was implemented, allowing users to quickly filter tasks using case-insensitive matching. Beyond algorithmic improvements, the user interface was enhanced with features such as priority-based color coding, which improves readability and reduces cognitive load. These enhancements demonstrate a strong understanding of how algorithmic decisions influence user experience, not just technical improvements.

Course Outcomes Alignment

The planned course outcomes for this enhancement were outcomes three and four, which focus on applying algorithmic principles and demonstrating effective use of data structures. These outcomes were successfully met through the implementation of sorting and searching algorithms, as well as the restructuring of the underlying data model. The project clearly demonstrates the ability to design efficient logic, consider time complexity, and apply appropriate data structures to solve real-world problems. At this time, no changes to the outcome-coverage plan are necessary, as the enhancement aligns well with the learning objectives.

Learning Experience

Throughout the process of enhancing this artifact, I have gained a much deeper understanding of how algorithms function in practical applications. One of the most important lessons I have learned is how to design multi-level sorting logic that balances efficiency with readability. Implementing custom comparators required careful thought about how data should be prioritized and compared. Additionally, integrating a real-time search feature introduced challenges related to performance and responsiveness, especially when filtering data dynamically as the user types. Another key learning experience was modifying the database schema to support new data fields while maintaining application stability. This reinforced the importance of planning for scalability and maintaining consistency between data models and application logic.

Challenges and Problem Solving Skills

There were several challenges encountered during the enhancement process. Ensuring that sorting logic worked correctly across different screens required debugging and testing multiple edge cases. The search functionality needed to be optimized to avoid unnecessary processing, and input validation had to be added to prevent invalid data from affecting the system. Despite these challenges, the process ultimately strengthened my problem-solving skills and reinforced the importance of writing clean, maintainable code.

Conclusion

Overall, this enhancement significantly improved both the functionality and usability of the application. It reinforced my understanding of how algorithms and data structures directly impact performance and user experience, and it strengthened my ability to design efficient, scalable solutions.

Enhancement Three: Databases

This enhancement strengthens data persistence, integrity, and security through improved database design and structured data management.

Download Enhancement Three ZIP

View Enhancement Three Narrative

Artifact Description

The artifact I selected for this enhancement is the To-Do List Application again, which was originally created during CS-360. The application allows users to create, edit, and manage tasks, with data stored locally using an SQLite database. For this enhancement, I expanded both the database and user interface to create a more complete and functional application. In addition to transitioning the database to a relational model with tasks and categories, I added a due date feature with a user-friendly date picker and full database integration. I also refined the application’s sorting logic to organize tasks in a more structured and meaningful way. These combined updates transformed the application from a basic task manager into a more realistic, fully featured productivity tool.

Reason for Selection

I chose this artifact for my ePortfolio because it demonstrates both my technical growth and my ability to enhance an existing system in meaningful ways. While the original version showed my understanding of basic CRUD operations, the enhanced version highlights more advanced skills such as relational database design, normalization, and SQL query optimization through JOIN operations. The addition of the due date feature strengthened the connection between the user interface and the database, ensuring consistent and accurate data entry through a calendar-based selection process. I also improved how tasks are displayed by incorporating due dates into both the main and categorized views. Additionally, I implemented a multi-level sorting system that organizes tasks by category, priority, due date, and alphabetically. These improvements ensure that tasks are always presented in a logical and user-friendly order and create consistency across different screens of the application.

Enhancements Implemented

This enhancement significantly improved both the database structure and user interface functionality. The application was upgraded to a relational database model that better organizes tasks and categories. A new due date feature was implemented using a date picker to improve usability and ensure accurate data entry. Sorting logic was also enhanced to support multiple levels of organization, including category, priority, due date, and alphabetical order. These changes improved both performance and user experience while making the application more scalable and realistic as a productivity tool.

Course Outcomes Alignment

Overall, I believe I met the course outcomes I initially planned, particularly in database design, structured data management, and application integration. Through this process, I developed a deeper understanding of how relational databases improve efficiency and organization, as well as how application logic can enhance the usability of stored data.

Learning Experience

One of the most valuable learning experiences from this enhancement was understanding how different layers of an application interact. I gained deeper insight into how relational database design affects application structure and usability. I also learned how to better integrate front-end components like date pickers with backend database systems to ensure data consistency and reliability.

Challenges and Problem Solving

One of the biggest challenges I faced was ensuring that all components of the application remained synchronized after modifying the database structure and adding new features. Implementing date-aware sorting required additional logic to correctly compare and order dates rather than treating them as simple text values. Maintaining consistency between multiple screens while updating sorting behavior also required careful testing and refinement.

Conclusion

Overall, this enhancement strengthened my confidence in building more complex, data-driven applications. It reinforced the importance of thoughtful design, incremental development, and ensuring consistency across all layers of an application. This experience better prepared me for real-world software development practices.
