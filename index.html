Enhancement One: Software Design and Engineering

This enhancement improves the structure, usability, and maintainability of the To-Do List application. Updates include refined sorting logic, improved user interface elements, and better code organization.

Download Enhancement One ZIP

View Enhancement One Narrative

The artifact I selected for this enhancement is the Android-based To-Do List application that I originally developed during CS-360: Mobile Architecture and Programming. The application allows users to create, update, and manage tasks through a mobile interface. It was initially created as a foundational project to demonstrate basic Android development concepts such as activities, layouts, and user interaction. For this enhancement, I revisited and expanded the application to improve both its functionality and overall design. I chose this artifact for my ePortfolio because it clearly demonstrates my growth in software development, particularly in mobile application design and architecture. The original version of the app was functional but fairly simple. Through this enhancement, I was able to build on that foundation by adding more advanced features such as task categories using a Spinner, persistent storage through a database, improved user interface elements, and input validation. These additions showcase my ability to refine an existing codebase, improve usability, and implement more structured and maintainable solutions. The inclusion of database integration and better UI design especially highlights my ability to think beyond basic functionality and focus on real-world application needs. This enhancement aligns well with the course outcomes I originally selected, specifically outcomes 3 and 4. I was able to design and evaluate a more complete computing solution by adding features that improve how users interact with the app and how data is managed. At the same time, I applied practical tools and techniques commonly used in mobile development, such as event handling, data persistence, and interface design. At this point, I do not feel the need to adjust my outcome-coverage plans, as the work competed clearly demonstrates these objectives. Working on this enhancement was a very valuable and fun learning experience. One of the biggest takeaways for me was understanding how to extend an existing application rather than starting from scratch. This required me to think more carefully about code organization, compatibility with existing features, and how changes in one part of the app could impact others. I also gained more experience working with user input validation and ensuring that the app handles errors gracefully, which is something that is easy to overlook in smaller projects. There were definitely some challenges along the way. One issue I ran into was getting the Spinner for task categories to function correctly, as it initially did not display any options. Troubleshooting that problem helped me better understand how adapters and UI components work together in Android. Additionally, integrating the database required careful attention to how data is stored and retrieved so that the app remained stable and responsive. Overall, I think this enhancement helped me move beyond basic app development and focus more on creating a polished, user-friendly, and maintainable application. It reinforced the importance of planning, testing often, and iterating on existing code, all of which are essential skills in real-world software development.
